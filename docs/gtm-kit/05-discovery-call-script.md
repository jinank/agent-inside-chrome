# Discovery Call Script (30 Minutes)

## 0) Goal

Identify one urgent browser-dependent workflow and decide whether to launch a 14-day pilot.

## 1) Opening (3 min)

- "Thanks for taking time. I want to understand one workflow that is still manual, quantify impact, and see if we can automate it quickly."

## 2) Current state (8 min)

Questions:

- Which browser workflow is most painful right now?
- Who runs it today?
- How often is it run?
- How long does each run take?
- What breaks most often?

Capture:

- Current process map (step by step)
- Systems involved
- Dependencies and blockers

## 3) Business impact (6 min)

Questions:

- What happens when this workflow is delayed or wrong?
- What metric is most affected? (SLA, bugs, throughput, conversion)
- What would "successful automation" mean in 30 days?

## 4) Technical fit (6 min)

Questions:

- Does the workflow require signed-in browser state?
- Are APIs incomplete or unavailable for key steps?
- Any strict constraints (security, data boundaries, audit)?

## 5) Pilot framing (5 min)

Propose:

- In-scope workflow
- Baseline metrics
- 14-day plan
- Success criteria

Close with:

- "If we can automate this workflow and hit the target metric, are you open to expanding?"

## 6) Post-call deliverable

Send within 24 hours:

- one-page pilot scope
- baseline assumptions
- timeline
- required inputs from their team
