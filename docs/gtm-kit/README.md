# RethinkSoft B2B GTM Kit

This kit is a founder-led B2B GTM operating system for RethinkSoft.

Use it in this order:

1. Define target customer and wedge in [01-icp-one-pager.md](./01-icp-one-pager.md)
2. Finalize narrative in [02-positioning-and-messaging.md](./02-positioning-and-messaging.md)
3. Package pilot in [03-offer-and-pilot-design.md](./03-offer-and-pilot-design.md)
4. Start outreach from [04-outbound-sequences.md](./04-outbound-sequences.md)
5. Run discovery with [05-discovery-call-script.md](./05-discovery-call-script.md)
6. Execute pilot with [06-pilot-sow-template.md](./06-pilot-sow-template.md)
7. Turn wins into proof with [07-case-study-template.md](./07-case-study-template.md)
8. Track execution using [08-metrics-dashboard-template.csv](./08-metrics-dashboard-template.csv)
9. Maintain cadence using [09-weekly-operating-rhythm.md](./09-weekly-operating-rhythm.md)

## Suggested 30-day rollout

- Week 1: ICP, messaging, pilot offer finalized
- Week 2: 100 target accounts, outbound starts, 10+ discovery calls
- Week 3: Launch first pilots, publish first mini proof artifact
- Week 4: Close first paid expansions and tighten GTM loop

## Initial recommendation

Start with this wedge:

- Buyer: Head of QA, Engineering Manager, or Support/Ops leader at B2B SaaS
- Pain: Manual browser-dependent workflows and flaky QA/research tasks
- Offer: "We automate one browser-critical workflow in 14 days with measurable ROI."
