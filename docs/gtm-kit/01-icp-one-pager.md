# ICP One-Pager

## 1) Primary ICP (start here)

- Company type: B2B SaaS
- Size: 30-300 employees
- Team maturity: Has product + engineering + support/ops workflows
- Buyer roles:
  - Head of QA
  - Engineering Manager
  - Head of Support / Operations
- Technical profile:
  - Uses web apps with authenticated workflows
  - Has repetitive browser tasks not well-covered by APIs

## 2) Wedge use case

Choose one workflow category only for the first motion:

- QA release validation in real browser flows
- Internal data extraction from authenticated portals
- Repetitive support operations in admin dashboards

## 3) Pain signals to qualify

- Team says: "We still do this manually every week."
- Automation attempts failed due to dynamic UI, auth, or brittle scripts.
- Browser workflow blocks release/support SLAs.
- There is a clear cost of delay (hours, bugs, response time, revenue impact).

## 4) Disqualifiers

- No repeated browser workflow
- "Nice to have" interest without urgency
- No internal owner for pilot execution
- No measurable baseline metric

## 5) Economic buyer and champion map

- Economic buyer: VP Eng / Head of Ops
- Daily champion: QA lead / Support lead / RevOps manager
- Security stakeholder: IT or security reviewer

## 6) Discovery fields to collect per account

- Current workflow owner
- Current process steps
- Frequency per week
- Time per run
- Error/failure rate
- Existing tooling attempts
- Expected improvement target

## 7) Business case formula

Estimated weekly hours saved:

`(manual_minutes_per_run - automated_minutes_per_run) * runs_per_week / 60`

Estimated monthly value:

`weekly_hours_saved * fully_loaded_hourly_cost * 4.33`
