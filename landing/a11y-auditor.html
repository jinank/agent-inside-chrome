Accessibility workflow

Audit the experience users actually touch.

Review keyboard access, semantic cues, contrast, labels, and focus behavior in a real rendered browser flow.

Keyboard

Trace focus behavior

Check tab order, visible focus, and keyboard-only operation.

Semantics

Inspect structure

Look for labels, role misuse, and assistive technology blockers.

Contrast

Find readability issues

Surface visual risks that affect comprehension and conversion.

Best for

Signup, checkout, dashboards, forms, and customer-facing workflows.

Inputs

Target page, key flows, expected user role, and known accessibility concerns.

Output

Prioritized findings with user impact, reproduction steps, and fixes to consider.

npx rethinksoft-in-chrome setup
